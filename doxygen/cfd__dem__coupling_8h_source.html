61  /**
62  * The cell_weight() function indicates to the triangulation how much
63  * computational work is expected to happen on this cell, and consequently
64  * how the domain needs to be partitioned so that every MPI rank receives a
65  * roughly equal amount of work (potentially not an equal number of cells).
66  * While the function is called from the outside, it is connected to the
67  * corresponding signal from inside this class, therefore it can be private.
68  * This function is the key component that allow us to dynamically balance the
69  * computational load. The function attributes a weight to
70  * every cell that represents the computational work on this cell. Here the
71  * majority of work is expected to happen on the particles, therefore the
72  * return value of this function (representing "work for this cell") is
73  * calculated based on the number of particles in the current cell.
74  * The function is connected to the cell_weight() signal inside the
75  * triangulation, and will be called once per cell, whenever the triangulation
76  * repartitions the domain between ranks (the connection is created inside the
77  * particles_generation() function of this class).
78  */
